# Updating the Website

The website at `website/` uses auto-generated data extracted from the Rust source code. The file `website/src/data/sourceData.ts` is generated by `website/scripts/extract-source-data.mjs` and must stay in sync with the codebase.

## When to Run

Run this skill whenever any of these files change:

| Source file | What it provides |
|-------------|-----------------|
| `zig-cli/Cargo.toml` | Version number |
| `zig-cli/src/cli.rs` | Commands, workflow subcommands, patterns |
| `zig-core/src/workflow/model.rs` | Step fields, variable types |
| `manpages/*.md` | Manpage topic list |

## Tracking Mechanism

The file `.claude/skills/update-website/.last-updated` contains the git commit hash from the last time the website was updated.

## Update Process

1. Read the baseline commit hash:
   ```sh
   BASELINE=$(cat .claude/skills/update-website/.last-updated 2>/dev/null || echo "")
   ```

2. Check if source files changed since baseline:
   ```sh
   git diff --name-only "$BASELINE"..HEAD -- \
     zig-cli/Cargo.toml \
     zig-cli/src/cli.rs \
     zig-core/src/workflow/model.rs \
     manpages/
   ```

3. If changes are detected, regenerate the source data:
   ```sh
   cd website && npm run extract
   ```

4. Review the generated `website/src/data/sourceData.ts` to verify:
   - Version matches `zig-cli/Cargo.toml`
   - All commands from `Command` enum are present
   - All workflow subcommands from `WorkflowCommand` enum are present
   - All patterns from `Pattern` enum are present
   - Step fields match the `Step` struct
   - Variable types match the `VarType` enum

5. If the extracted data changed, also review the website components that reference it:

   | Data field | Used by |
   |-----------|---------|
   | `version` | `Hero.tsx` (version badge) |
   | `commands` | `Features.tsx` (count), `GettingStarted.tsx` (reference) |
   | `workflowSubcommands` | `GettingStarted.tsx` (reference) |
   | `patterns` | `Features.tsx` (count), `Patterns.tsx` (cards) |
   | `stepFields` | `Features.tsx` (count), `WorkflowFormat.tsx` (reference table) |
   | `varTypes` | `WorkflowFormat.tsx` (description) |

6. If new commands or patterns were added, consider:
   - Adding new terminal demo sequences in `website/src/data/terminalDemos.ts`
   - Updating feature descriptions in `website/src/components/Features.tsx`
   - Adding pattern examples in `website/src/components/Patterns.tsx`

7. Build the website to verify:
   ```sh
   cd website && npm run build
   ```

8. Update the tracking file:
   ```sh
   git rev-parse HEAD > .claude/skills/update-website/.last-updated
   ```

## Implementation Files

### Auto-generated (do not edit manually)

- `website/src/data/sourceData.ts` — Generated from Rust source

### Extraction script

- `website/scripts/extract-source-data.mjs` — The extraction logic

### Website components that consume source data

- `website/src/components/Hero.tsx` — Version badge
- `website/src/components/Features.tsx` — Feature cards with counts
- `website/src/components/WorkflowFormat.tsx` — Step field reference
- `website/src/components/Patterns.tsx` — Pattern cards
- `website/src/components/GettingStarted.tsx` — Command reference
- `website/src/data/terminalDemos.ts` — Terminal demo sequences

### Editorial content (not auto-generated, may need manual updates)

- `website/src/components/HowItWorks.tsx` — Three-step flow
- `website/src/components/BuiltOnZag.tsx` — Architecture diagram and comparison
- `website/src/data/terminalDemos.ts` — Terminal animation sequences

## Verification

1. Run `cd website && npm run build` — must succeed
2. Verify `sourceData.ts` version matches `zig-cli/Cargo.toml`
3. Verify command count matches the `Command` enum variants
4. Verify pattern count matches the `Pattern` enum variants
5. Confirm `.last-updated` was written

## Skill Self-Improvement

After completing an update session, improve this skill file:

1. **Add new source files**: If new Rust source files provide data for the website, add them to the extraction script and this mapping.
2. **Add new components**: If new website components consume source data, add them to the component mapping.
3. **Record patterns**: If you found a recurring update issue, document it here.
